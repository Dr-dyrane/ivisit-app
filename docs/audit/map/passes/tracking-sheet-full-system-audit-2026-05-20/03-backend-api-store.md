| Patient cancel/complete lane | `useEmergencyHandlers()` calls `setRequestStatus(cancelled/completed)`, updates visit lifecycle, and runs store cleanup only after the request batch succeeds.                                                        | Failed patient actions should preserve tracking state but must show failure copy, because backend rejection is a normal safety outcome.                |
| Arrival/check-in lane        | `onMarkAmbulanceArrived()` and `onMarkBedOccupied()` locally block statuses outside `in_progress/accepted` before calling the patient RPC.                                                                            | The UI has a preflight guard, but realtime can still make the CTA stale; returned `{ ok: false }` must become visible feedback.                        |
| Staff terminal lanes         | `complete_trip`, `cancel_trip`, `discharge_patient`, and `cancel_bed_reservation` have role/org/service checks and can terminalize the same request rows.                                                             | Patient tracking must treat staff terminal realtime/query updates as authoritative even if the patient did not press cancel/complete locally.          |

Fix shape: keep patient tracking actions on canonical constants and canonical request UUIDs; expose backend rejections as user-visible toasts while preserving current local state. Treat staff/console terminal events as external truth that should close/suppress live chrome through realtime/query/lifecycle, not as patient action failures.
